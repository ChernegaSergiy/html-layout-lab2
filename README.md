## УЧАСНИК 1

### Завдання для першого учасника:

| Завдання | Команда Git |
| :--- | :--- |
| Створити файл **`index.html`** та додати обов'язкову структуру: `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`. | - |
| Додати **семантичні теги** `<header>`, `<nav>`, `<main>`. У `<header>` розмістити заголовок **`<h1>`** та **навігаційне меню** з внутрішніми якорями **`<a>`**. | - |
| Створити першу секцію **`<section>`** у `<main>`, яка містить **інформацію про Учасника 1**: заголовки **`<h2>–<h6>`**, абзаци **`<p>`**. | - |
| Виділити важливі слова **`<strong>`** та **курсивом `<em>`**. Додати зовнішнє посилання **`<a>`** та вставити **зображення `<img>`** з атрибутом **`alt`**. | - |
| Перевірити статус файлів | `git status` |
| Додати файл під версійний контроль | `git add index.html` |
| Зробити перший коміт | `git commit -m "Uчасник 1: Ініціація, структура, навігація та інформація про себе"` |
| Запушити зміни до віддаленого репозиторія | `git push origin main` |

## УЧАСНИК 2

### Завдання для другого учасника:

**Перед початком роботи:**

| Завдання | Команда Git |
| :--- | :--- |
| Отримати останні зміни з віддаленого репозиторія | `git pull origin main` |

**Основні завдання:**

| Завдання | Команда Git |
| :--- | :--- |
| Додати нову секцію **`<section>`** (або `<article>`) з **інформацією про Учасника 2**. | - |
| Створити **нумерований `<ol>`** та **маркований `<ul>`** списки (наприклад, навички та досягнення). | - |
| Перевірити статус файлів | `git status` |
| Зберегти змінений файл | `git add index.html` |
| Зробити перший коміт | `git commit -m "Учасник 2: Додано профіль та списки"` |
| Додати нову секцію. Створити **таблицю `<table>`** (3–4 стовпці, 4–5 рядків), використовуючи теги **`<tr>`**, **`<th>`**, **`<td>`** (наприклад, розклад). | - |
| Додати секцію **мультимедіа**: вставити **аудіо `<audio controls>`** та **відео `<video controls>`**. | - |
| Перевірити статус файлів | `git status` |
| Додати файл в індекс | `git add index.html` |
| Зробити другий коміт | `git commit -m "Учасник 2: Додано таблицю та мультимедіа"` |
| Запушити зміни до віддаленого репозиторія | `git push origin main` |

## Примітки:

1. **Учасник 1** створює та ініціює репозиторій, **Учасники 2 та 3** клонують його.
2. **Усі учасники** повинні виконати конфігурацію Git зі своїм ім'ям та email.
3. Учасники виконують завдання **послідовно**: спочатку Учасник 1, потім Учасник 2, потім Учасник 3.
4. **Важливо**: Кожен учасник повинен отримати останні зміни (`git pull`) перед початком своєї роботи та запушити зміни (`git push`) після завершення.
5. Репозиторій повинен бути створений на **GitHub** для можливості клонування.
6. **Назва файлу:** Рекомендовано використовувати `index.html`.
